## 整备：高考结束后的暑假

废话不多说，首先看 ELE 实验室的模数电基础（analogue，digital），最多就是高中物理的知识要求，甚至不需要，你这下知道除了二极管之外还有三极管，你可以通过控制栅极的电压或者电流来把控另外两端之间的导通与阻塞，这是一个电子开关，用数个这样的开关组合，你会得到与非门，这是万能且基础的逻辑门，逻辑门层面的原子，这是第一层封装。使用与非门可以构建其他逻辑门。只需要记住，和电源相连的引脚电压等同，接地的电压则为 0，这就是 0/1 的二进制表示。逻辑门进而能够组成复杂的数字逻辑电路，这是第二层封装。

紧随其后的是 crash course 系列的经典，计算机速成课，它总览了计算机科学与工程，从逻辑门到第一个加法器，从逻辑门到一个内存 bit 构建一个 block 再到一个 bank.... 加法器和其他运算单元进一步构成 ALU，你还需要额外的电路将数据加载到 ALU，CPU 初见端倪。视频里会反复提到，要学会最重要的思想：**封装和抽象**。你马上可以用 logism（仅几 MB 的数字逻辑仿真软件，free of charge），from zero to one build cpu 造一个自己的 8 位 cpu，当然这是非常原始且外围 io 几乎为零的裸片，你可以试着加上各种外设：显示器、硬盘、....。外设多了就需要考虑 CPU 和它们的交互，接口设计的可扩展性（如果要增加更多的外设，能否在不修改 CPU 电路的前提下实现？），你会想到使用外设控制器，这便又是一种封装和抽象。

crash course 系列的课程都是精品，每节课都只有十分钟，但是却能让你无痛入门任何一个领域。特别推荐《哲学》和《科学史》系列，你要看着前人是如何一步步到达这里的。在奥德赛的年纪，每一天你的思想都在发生剧烈的变化，人生观、世界观和价值观皆酝酿这风暴。这是人生尺度的话题，但你的每一天都在变得越来越快，人的感受是指数变化的，按照每一天在剩余生命里的百分比来计算，你人生的中点是 18 岁。

ELE 实验室的创作领域集中在广义 cs 上，动画精美深入浅出，可以持续关注。这部分结束后，你应该对 cpu 的构造有宏观的理解，同时也能在微观上"脑内执行"每一条指令，你知道数据和指令分离的冯诺依曼架构已经逐渐和哈佛架构融合。当然，现代 cpu 拥有更复杂的多核交互、流水线以及非标量超线程的设计，你可以按照兴趣进一步探索这颗人类科技皇冠上的明珠。

CPU 只是现代数字自动化基础的一环，你肯定听过 GPU，DSP（数字信号处理器，包含快速傅里叶变换、浮点运算单元和定点运算单元以及用于函数查找的表），NPU（神经处理单元，用于运行特定的神经网络），XXU。他们在逻辑控制、成本、并行上有不同的优势，根据需求应用于各行各业。如果走量，会选择 ASIC（Application specific integrated Circuit），这相当于把代码硬化在逻辑电路中；如果大量但没那么大，FPGA 是更好的选择，它内部包含了多种常用的功能块以及可以可编程（可以灵活地转变为各种各样的逻辑电路）的逻辑块，FPGA 逻辑资源的多少决定了它的"变形能力"。

以及如果你想在简单的 cpu 基础更进一步，我推荐你阅读《程序是怎样运行的》这本小册子。你说老师，我太想进步了，那你再看看计算机组成原理透视版，以及 csosc。还是不过瘾？必须要重拳出击了：《Computer Systems: A Programmer's Perspective》，CSAPP。想要更多吗，真是的，哈基，你这家伙…. : [csdiy.wiki](https://csdiy.wiki/) 包含了从犄角旮旯翻箱倒柜汇集而成的 cs 资源。你陆续在探索的过程中发现了更多的神人，比如小林 coding，计算机系统和计算机网络里分门别类放好的可视化把算法通通具象，你感叹要是自己早点知道这里就好了。前人栽树后人乘凉，等你退休了一定也要搭建一个这样的网站帮助后来者。

如果你的 cpu 造得差不多了，便可以看看 Python 这个新手友好的编程语言。它的语法规则和英语差不多。当我们在说"语法"的时候，我们在谈论什么？别担心，如果你有幸学习编译原理的话就会亲自揭开谜底。你只要知道 Python 运行起来不需要经过编译，只需要一个小巧的解释器就可以把表达式翻译成 cpu 认识的二进制，你一定已经清楚指令集、汇编语言和微代码的区别了。python 有很多 oneliner（所有功能在一行实现），但这并非最佳实践，你之后会知道什么叫"设计模式"。Python 最好的教程是廖雪峰，语法可以看 runoob。Python 的开发环境很好配置，只需要下载安装包即可，在 linux 下仅需要给包管理器一条指令，其跨平台特性是很大的优势之一。你不应该在记事本或者 word 里编写代码，遑论手抄代码。你会听说 VSCode，若你对其 UX（用户交互界面）一无所知，不知道从何开始的时候，官方文档是你最好的伙伴。PyCharm 同样可以作为有力候选人，它的专业版现已对学生免费，这是为了培养用户粘性扩大市场。怎么申请？你需要一个 .edu 教育网后缀的邮箱。如果你现在没有，别担心，入学之后该有的都会有。这些 IDE（集成开发环境）有着丰富的插件生态，点开插件市场筛选 popular，看看有哪些适合你。在学的过程中一定要边看边写，掌握基本语法之后可以尝试 python 小项目，比如抓取天气预报，自动打卡，写一点脚本；你还可以通过一些库控制电脑，调用操作系统接口，甚至访问其他应用程序。你肯定听过 pytorch 或者 tensorflow 的大名，但先别急着用它们训练神经网络，该来的不会少。你现在的当务之急是，掌握 python 内置的各种数据结构，搞明白 python 的函数传参是引用还是拷贝。如果你能脸不红心不跳地写出列表推导式替代循环，用矩阵运算换掉迭代，自如地编写递归程序时，便可说是大功告成。

在 import package 的时候你发现有时候会找不到包，经过一番挣扎，搜到了结果，你用上了 pip，还学会了更换软件源镜像，清华软件源让你一见倾心。更美妙的方式是使用 virtualenv，也可以选择 AnaConda/MiniConda 以及 pixi，uv 等候选者，它们都提供了强大的隔离和封装，以便在其他平台、其他 python 版本也可以一键配置。VSCode 贴心地集成了这些包管理器地 GUI，只需动动鼠标点两下便可准备完成。你想尝试一下 Ubuntu 下的 python 配置，但发现有时候 WSL 和 windows 的环境变量会打架，在别人还在好奇环境变量是什么意思的时候，聪明的你通过配置 **wslconfig** 轻松解决了这个问题。版本兼容性的地狱还没有降临，你需要亲自体会在 Ubuntu25.04 下使用 ROS1 的绝望——耶和华，第六章第七约，我编的。**RoboStack** 将会处理这一切。你再次体会到了封装和抽象的好处，你暗下决心，我一定要学会世界上所有的虚拟化工具。别急着凑上前看 RoboStack 内部是如何实现的。

此时此刻恰如彼时彼刻，VSCode 右下角跳出了一行提示，Copilot Edit is ready for you, don't fly alone。这是一个危险的信号，就像 matrix 里提醒 neo：follow the white rabbit。Vibe coding 和其他编程辅助工具是潘多拉的魔盒，强大且危险，恐怕你的功力还无法驾驭它，暂时放一放吧，我们会在恰当的时机引入它。如果你都无法构思一个功能该如何实现，又谈何让别人帮你实现？"己所不欲，勿施于人"，没关系的。少年，继续筑基吧，系统已经启动。

也许你发现自己的打字速度有一点慢，以前别人总是叫你峡谷钢琴家，但现在你被发配到了祖安。那么你一定要去 typingclub 或者 typing 练习标准指法。键盘上 F 和 J 键的小凸点就是帮助你定位食指的。qwer 布局是否落入俗套？或许 Dvorak 和 Colemak 是更好的选择，它们拥有更合理的左右手负载均衡。写代码的时候右手常常在鼠标和主键盘区切换让你烦恼不已，恰到好处你了解到 **VIM**，天呐怎么会有这么好的设计，模块化的功能，丰富的插件，强大的命令模式以及 Unix 的"只做一件事并把它做到最好"的哲学深深折服了你（现在，立刻马上去学习 vi！）。你认为鼠标就不应该被发明出来。不过我是 emacs 的忠实粉丝。neovim 已经提供了 vscode 的插件集成。记得保存编辑器的配置，不然要远程连接到服务器上可就大事不妙了。

电脑并非只用于编程，你最好了解常用软件的 UI 和 UX，思考它们为什么要这么排布，"人体工学设计"映入眼帘。"好的设计应该给用户最小的 culture shock"，你在笔记本中写道。你花了点时间概览常用的办公软件，在 bilibili 搜索栏里键入"windows 设置，windows 性能调优，windows 必备软件，Linux 不这么设置就可惜了，MacOS 装机必备不装相当于白买……"。好了，注意不要矫枉过正。你又装上了几个趁手的小工具，包括用于快速查找文件的 everything，预览文件的 quicklook 和强大的快捷宏 quicker，以及 powertoys。你寻思应该搞个 **ladder**，去看看外面的世界。我们需要站在巨人的肩膀上。如果你是中外合办、港校或者海本爷，那么恭喜你，你已经赢了一半了（在起跑线？）。

相信 ai 这个词已经你的脑子洗了三十遍，目前这些模型还是偏科生，而你是一张白纸。暂时把 llm 当作能通过自然语言交互的搜索引擎和插件增强是个不错的视角，它是一位孜孜不倦的老师，不会嫌你菜也不会摆烂，只是稍微有点费电。好多企业都推出了自己定制的"AI"，现阶段你可以把 AI 当作多模态大语言模型的同义词，把他们的 app 都下载下来玩一玩，它们专注解决的问题不同，可以按照指示生成、修改图片，可以制作 PPT 大纲，可以和你通话练习口语，可以写小说，总结摘要、润色文章。你知道 prompt 是什么意思吗？字如其名，提示词目的是：在输入问题和其他内容之前，告诉 LLM 它将扮演何种角色、以何种形式返回自己的回答、具体需要完成什么任务、可以动用何种资源……你会看到很多 prompt 的设计里都有尖括号：<>，这是点化模型的咒语。大模型和人阅读信息的过程不太一样，他首先通过分词器把一句话分成一个个 token（查查这是什么意思？），然后再对 token 进行处理。它的输出结果也是 token，最终通过字典翻译成人类可以理解的语言。token 是大模型内部的通货。

prompt 是让大模型正确发挥能力的咒语，甚至有人以设计 prompt 为生。记录一些常用 prompt 或者创建模板可以极大地提升你的工作效率。但千万不要过度依赖 LLM，务必先进行自己的思考而不是选择让模型"深度思考"，不要让大模型变成你的"自然语言编译器"。你也要意识到 llm 能力的边界，"幻觉"困扰着你，更困扰着无数 LLM 算法工程师。你必须拥有鉴别模型输出的能力，这要求你对问题本身有一定的理解，否则你甚至无法写出正确的 prompt，不知道该如何组织你的问题。因为"人无法想象未见之物"。尝试给出一些思路，把你的方案和思考过程记录下来，提供给 LLM 甚至能极大地提升它的回答质量。

闲来无事刷刷社交媒体不算什么大过。关注一些企业的宣发号是不错的选择。我会推荐你关注神经现实、集智俱乐部、软件智库和中科院物理所。bilibili 也有好康的：IT 咖啡馆，科学史（划重点后面要考）。几个 IT/CS 动画人类高质量：代码坊译制，branchEDU，技术流，硬件茶谈，织点代码，nomaker，geekhour，efficentLab，漫士沉思录，geekbay（你现在必须能看懂他的视频了！），大名鼎鼎的 3b1b，freeCoder，Core Dumped（不少博主搬运翻译的都是他的视频）。

你肯定记了不少笔记，不过亟需一个数字大脑帮你更好地组织碎片化的单页。Notion（可用 AppFlowy 作为开源替代），obsidian，Flowus，印象笔记，飞书文档等都是不错的选择，不要纠结太久，它们基本都支持主流的富文本编辑（markdown plus！）和双向链接。选择一个大概率不会倒闭的就完事了。具体怎么建立自己的知识体系？小试牛刀，用你刚刚学会的技巧去信息的海洋里遨游一番吧：找到适合你的方法论，知乎、少数派和简书等平台上都有不少分享此类体系的博主。这是终身受用的技能。

### 数学

数学的学习仍然是老三样，不过微积分，线性代数，概率论可以一起学。以应用为目的，避免陷入空中楼阁的尴尬情景，务必按照人类的学科发展历史顺序阅读，以符合人类认知阶梯的方向引入，而不是直接使用定义。普林斯顿微积分读本系列教材，Gilbert Strang 的微积分 essence 都是很好的范本，国外的教材和课程都会先强调直觉再进行推导。知乎上鞭笞同济数学绿皮和俄系教材的帖子很多，这里就不借花献佛赘叙了，自行搜索即可。BTW，知乎直答是个做的不错的 RAG LLM。对于数学学科发展，推荐阅读《西方文化的数学》，《数学简史》，以及长河劫的科学史系列视频。课还是要上的，以应试为目的的学习，微积分课程推荐国防科技大学朱健民教授的 mooc 课程。

虽然初中高中都反复强调学习需要内驱力，之后没人教你怎么学，但实际上填鸭仍在继续。要跳出被动学习的怪圈你首先要接受自学时难以进入心流的窘境。扩展新领域时被不熟悉的感觉充斥是正常的，你很难进入完全专注的状态，甚至为此感到痛苦，注意力涣散，被周遭各种平时认为百无聊赖的事物吸引。新的模式对于以前的行事准则冲击很大，接受这种反差即可。如果你开始感到轻松，那么需要加快速度更进一步，踏出自己的舒适区，这是危险的信号，说明你已经开始对它感到熟悉。你要利用好对未知的恐惧。

### 编程与工具

培养计划里必然搭配了计算机导论 + python/C 二选一的组合。学习 c 语言，推荐使用《C语言经典教程》，C++ 推荐会教你敲 `g++` 指令的翁恺老师：C++ 面向对象程序设计。别在开发环境的选择上犯难了，犹豫即是死亡。devc++ 编译器不用自己配置，Visual Studio 也一样开箱即用。为什么不试试把在 vscode 上搭建环境，现在你会怎么 onboard？别忘了官方文档永远是第一选择，迟疑一秒都是对最好的资料的不尊重。务必学会自己通过 gcc/g++ 传递参数、手动编译多个源，包含头文件引入库路径，编写用于编译的 json/shell 脚本做定制化，再到 makefile 和 cmake。现在有数不胜数的构建工具，cmake 目前还是无法逾越的事实标准，当然你可以玩玩 bazel，xmake 这也无伤大雅。如果你在编写大作业的时候用到了第三方库和自己之前写的模块，你会意识到一款好的包管理器对于大型项目开发是至关重要的，你了解到 windows 环境下除了 msbuild 和 msvc 还有 mingw，msys2，cygwin。当然你也可以选择使用 winget 和 vcpkg。

### AI 辅助编程

听着，是时候打开潘多拉的魔盒了，你现在终于可以构造托卡马克引擎，把魔法限制在指定的区域，不允许产生任何泄露！鉴于你已经有计算机体系结构和操作计算机的咒语：Python 和 C++ 的基础，相信你已经有解决任何初级问题的能力了。尝试使用 Copilot 来帮你减轻重复劳动的负担吧，但你要先保证自己有能力破解这些难题——你只是懒得做罢了。这是迈向自动化的第一步。你很快会沉浸在 LLM 强大的上下文能力，拜倒在 copilot 女神的石榴裙下。怎么才能让 tokens per seconds 更高一点？

### 嵌入式入门

玩玩 arduino 和 51 单片机怎么样，这是精进 C/C++ 最好的方式。你可以直接把控 IO 操作硬件，管理 DMA，你就是操作系统。这对于理解内存模型以及复习计算机的工作原理很重要。可以做个蓝牙遥控小车，玩玩串口转蓝牙模块、循迹和超声波模块，用电桥驱动电机，翻转 io 控制 mos 管。你肯定知道数据在模块间的流动，各类总线如何设计通信协议。现在你会怎么找资料？把手机上的操作到小车响应的每一层抽象都想清楚，每个层次的模块应如何看待、处理数据的，想想为什么要有流的概念，从不同的视角剖析这一切。和你最爱的 LLM 一起讨论到天荒地老吧，也许交换彼此的角色，使用费曼学习法和小黄鸭调试法会有别样的体验：周舒畅的 LLM 讨论课堂、超级提示词工程。

在玩单片机的时候，你开始关注一些论坛，stack/mathoverflow，电子发烧友和 eeworld，21ic，阿莫电子论坛，stm32community（官方论坛和 github issues 永远是你的快乐老家）、稀土掘金、思否等等，质量最高的自然还是栈溢出。Github 真是个宝藏，里面的人说话又好听，超喜欢在里面的。什么，你还不知道怎么用 git？快看看官方的 workshop 和文档吧：Git Tool Get Started，图形化 git 示例：learn Git。但是东西太多根本看不过来怎么办？是时候搭建自己的文档解析服务器了，尝试设置本地或者服务器端端 lobechat or 使用现成的商业产品，取决于你的时间。有了自动化工具的帮助，把大量文档编码为向量数据库，将 LLM 调教成问啥答啥的 chatbot 易如反掌。你已经不满足于 8 位和 32 位的机器，欲求在 64 位的 cpu 和嵌入式 linux 里寻找更刺激的挑战，RTThread 和 ThreadX、FreeRTOS 的任务调度和线程通信对你来说已经是小菜一碟。你可以随手列举 10 种通信的手段和临界区保护手段，你的名言是，操作系统就是保存现场，切换上下文再恢复现场罢了，这有什么大不了！

### 社团与竞赛

学校的社团招新了，我们称之为百团大战。你在兴趣爱好之余可以多关注科协、计算机协会、创客、机器人竞赛相关的团队。我推荐你加入学校的 RoboMaster 战队，他们说这是全球青年工程师的最高荣誉殿堂，我不置可否，你得自己去看看。你可能会遇到 35 个和你一样强的本科生，他们甚至在高中的时候就已经干了 vex 和 FRC，你为此感到丁寒。放手去做吧，不用担心被嘲笑，被误解，你从来不需要从他人的口中获得认可。Robocon、智能车和工训大赛都是不错的去处。Robocup 下属的赛项很多，如果你的学校做的不是足球机器人，或者直接从浙大购买成品，那你要审慎考虑加入了。

### 机械设计基础

工科大类大一还会学习工程制图或者机械制图，但你不要用老掉牙的 autodesk 2d 制图。并不是说 autodesk 不好，只是大部分 cad 软件都是 3d 的，这更符合设计直觉。FDM 打印机已经非常成熟，你可能听说过拓竹在消费级 3d 打印机行业大杀四方的故事，这是一支来自 DJI 的创业团队，DJI 常常被称作消费电子的黄埔军校，这绝非戏言。3D 打印机方便快速的 demo 迭代，我推荐 sw 或者 catia，你最好没有罹患选择恐惧症，选一款然后一直用下去，它们的区别不大。这些软件一般都有官方提供的示教例程，跟着它一步一步操作就能学个八九不离十。一开始你只会用到建模的功能，学会用减材的思想画图，把你的鼠标当作 6 轴加工中心的铣刀。你要学会圆倒角，设置粗糙度和公差带，合理设计轴系配合，制作装配图，然后将这些小玩意在 FDM 里做出来。常用的轴系连接，运动副的自由度计算必须烂熟于心。推荐知乎机械设计。了解加工相关的概念，公差和配合不需要背但要知道有这么个事，什么类型的材料用怎样的工具加工，材料的力学特性，各种标准件的用途。安装顺序也同样重要，设计的时候要考虑很多。机械设计是一个吃经验的领域，知乎上有很多博主分享自己积累的宝贵经历，比如：wongpangfaye；另外推荐两本使用工具书：日本经典技能系列丛书的《操作工具常识及使用方法》和《机械样图解读》。当然，除了 3d 打印，常见的车钻铣床冲压锻镗电火花滚花磨及其对应的工具和刀头选择也需要知道个大概。类似 [thingiverse.com](https://www.thingiverse.com/) 的网站上有广大网友共享的各式各样的手办和小工具模型，下载下来切片之后就能打印。

减速箱是个典型的例子，机构设计的巧思如无级变速可以极大地减少控制工程师的工作负担，往前追溯还有微分分析仪这种精密机械的集大成者。航空发动机迟迟没有突破除了材料上有差距，结构的设计同样会影响扇叶的模态、涵道的共振频率这些参数。机械结构确定之后，下游控制能做的事情很少，因此控制工程师往往要一同参与设计，给机械工程师提出指标。更多的机械相关知识参考知乎，这里抛砖引玉，肚子里没多少墨水就不丢人现眼了。

### 生涯规划与通识

开学之后，班会、年机会和开学典礼有很多生涯规划、学长学姐的考研保研经验分享。选择性地听听就好。诚然你的确需要尽早对自己的职业和学业进行规划，但你周围有太多的声音，很容易迷失在分岔口中郁于抉择。你必须找到自己的兴趣然后坚持下去。

这时倘若你还能保持阅读的好习惯，那我愿称你为时间管理的神。在这个短视的年代，大家都沉迷于唾手可得的多巴胺里，很多人一两年都不会读一本书，甚至没法静下心看一场不那么高潮迭起的电影。你要继续背单词，直到词汇量与母语者的高中水平相当。你很快会意识到英语有多重要，不要将其当作负担，而是为你所用的工具。世界上有 95% 以上的网页和文献著作选择了英语，这是预言成为现实，你只能接受。换个角度想，英语作为一个舶来语的缝合怪，语法相比"正规"的血统来说简单太多，你不用担心多如牛毛的变格和阴阳。双语使用者会切换不同的脑区，这会延缓你大脑的衰老，并且激活不同区块的语言中枢。注意使用英语的方式思考，而不是在母语和英语间来回翻译，你的大脑没有那么多的缓存供你使用。

你这四年会上不同数目的"水课"，包括但不限于形势与政策、思想道德修养和近代史等等。八仙过海，各显神通，用最短的时间以最优的边际效用把它们处理好。大部分课程不会计入核心课的加权之中。有没有用？这是仁者见仁智者见智的环节了。但我还是建议你好好读读《资本论》，也许是 21 世纪修订版、解读版。如果有一门课叫实用写作并且有英语版本的话，还是可以为之花费一些时间的。英语写作和中文写作在形式和叙事逻辑上都有较大的不同，看一些范文，掌握写作的模式可以加速英语的学习，同样有利于之后论文的阅读。

此外，不要疏于体育锻炼。很多同学仅仅过了半年运动能力就出现了显著下降。学习是一场长征般的修行，并不是打坐看书就能无伤练级飞升的。尤其注意心肺功能的锻炼，尽量不要熬夜，戒烟戒酒。你知道的，相关关系不等于因果，但你应该把这些可能性都扼杀在摇篮里。之前的疏忽对身体造成的损伤会以各种各样的形式加倍地返还给你，对精神的冲击同样不可忽视。要么在困意中永远沉睡，要么在痛苦中祈求处决。定期做全身的体检，校医院的检查基本是免费的。给自己买个商业险费不了多少钱。如果你的膳食不够均衡，学习如何合理地摄入膳食补充剂总是有益无害。

### 寒假：信息整合与个人项目

一个充实的学期过去了，寒假的时间很短，你觉得一切都是可以探索的，多逛逛知乎、不要被困在信息茧房里，是时候扩充自己了解世界的途径了，你需要更多的耳目。RSS 订阅、medium、Quora、Reddit，当下的世界每天产生的数据量规模超过 491EB（10 亿 GB），在这么大的尺度下你已经丧失了对"巨大"概念的感知，该使用对数坐标了。不如问问 chatbot 你应该怎么办吧，似乎没有人可以抵御信息洪流。诉诸于专业机构的报告和行业数据库可能是个不错的选择，这何尝不是一种封装和抽象？嗨，不如行动起来，把自己能够动用的信息源进行整合，丢到 LLM 里帮你总结，推送到邮箱每天花个几分钟看一看？这个过程涉及到计算机网络、信息收集与分类、推送服务器搭建、跨进程跨主机的通信协作，是个不错的练手项目，把它当作你的第一个系统级别的作品吧。你需要了解 web 三件套：html/css/javascript 构建交互的前端，可以使用现成的框架简化你的工作（抽象和封装！），后端的主力仍然是 C++，你或许想试试 GoLang 和 Rust，切记，不要贪多。掌握更多语言并不会让你变成更好的程序员。你还可以让 LLM 教你如何使用数据库，让一种语言调用另一种语言，再通过 python 一类的胶水语言把整个框架缝合起来。挑战自己 commit 的极限吧。虽然已经有不少低代码甚至零代码的商业产品可以实现跨平台的数据治理，但你仍然需要亲自走一遭以夯实对它们的理解，部署管理工具可参考此回答，购置一台工作站（捡垃圾～）或者在服务器上支起来都可以选择。在动手之前，使用 UML 绘制一个系统的框架图，并标注数据流，draw.io 和 ProcessOn 会是不错的帮手。编写并导出 doxygen 文档保证代码的可复用性和可维护性，你当然可以让 LLM 帮你完成这一切，甚至能为你生成单元测试。这么大规模的项目，别忘了使用 CI/CD，github 已经将这些功能集成，花几分钟学学 action 文件的编写。你肯定还希望将不同的功能模块封装成即插即用的形式，用垂涎已久的虚拟化技术将他们打包吧。

继续坚持记录自己的笔记，做好分类，正反链。把你遇到的 bug、设计的原型图和 BOM，在原地打转时脑袋里的一团乱麻、开悟前的瞬间记录下来，这将会一笔宝贵的数字财富。遇到类似的事，当别人需要上手未曾踏足的领域，你会如何为他们规划学习路径？

信息量太多，渠道太多。想想办法自动化 daily routine，比如 notion 提供了同步块、集成和连接来聚合不同平台的数据更新，你可以利用现成的插件，也可以自己写一个小插件。你的建立知识体系已经抽枝生芽，碎片化地记忆对你来说早已成为过去式。除了 markdown 这个程序员的最爱，latex（introduction to LaTeX），typst（next generation latex）也可以用起来了，漂亮的公式和排版让人身心愉悦。typst 的动态实时渲染让他成为 latex 的有力竞争者。你可以在本地建立 tex Studio 自己完成编译，也可以选择在线的成熟方案 overleaf，quiver 可以绘制 commutative graph，TikzMaker 支持 tikz code 和图像之间的双向转换，Mathcha 则可以绘制各种图形、使用宏编写 latex 代码并导出 tikz code。

自然，建立个人主页对你来说不在话下，github 上键入几个关键词，在 copilot 的帮助下几分钟内一个动态网页就成功上线了。写一些你个人的见解，在某天某人也许会通过藏在白色字体后的邮箱联系你，你们将结成很棒的队伍继续前行。不用过于纠结 js 和 css 里面花哨的 tricks 和 over engineering 的部分，你很快会迈向下一个驿站了。

最后，推荐一本培养兴趣的小黄书：《利己主义的数学解析》，从数学的视角为生物学、社会学和进化心理学的读者介绍了运筹和博弈的基本概念，以及导出的有趣的结果。你隐隐觉得这背后藏着不可告人的惊天秘密，可惜你终其一生大概率只会"get used to math, instead of understanding the math"。别担心，这是常态，大部分人如此。

你在机器人社团里的培训也应接近尾声，你接受了不少机器人通识课的学习，知道机器人一般有感知决策规划控制四个模块，也了解了端到端的兴起。你对机器人学的兴趣空前高涨，励志成为机器人系统工程师，开辟属于自己的一番天地。

### 线性代数深入

你在数学的学习上已经略微超了进度，把其他也在学习线性代数的同学甩开在身后一小截。不过大多知识是浅浅过了一遍有个印象，如果你不使用它们，它们将很快与你作别。学校的课程开始讲授多元微积分，你也该正式学习线性代数了。线性代数是几何与空间变换的艺术，我拾人牙慧，和其他人一样推荐你看 linear algebra done right 和 done wrong，以及它们的究极缩减版：linear algebra abridged。线性代数为何如此重要在此不大费周章介绍了。看过并不代表你真的懂了，很多时候这些提供直觉的教科书以及 3b1b 类似的天花乱坠的可视化视频让你有一种幡然醒悟的感觉，你需要趁热打铁做上两道题好让自己更加确信，你的确有几分深思了。线性空间将会是你接触的最后一个能够形成直观几何认识的数学分类，进入高维和非线性的领域，极度抽象的空间与代数霸占了那里的一切，你得谨慎起来了。线性代数的核心是"变换"，把四个子空间牢记于心，再加上矩阵分解的些许调料，你便师成出山。如果别人问起对偶和伴随，你答还没学便是，在优化、泛函和变分里，我们还会再见面的。

### 概率论与编程进阶

言归正传，有了一元微积分的基础，先别管勒贝格积分黎曼积分这那的，直接开始提前学习概率论。这大概率（基于贝叶斯推断和频率推断）是大二上的内容。别担心，它们就是就是把离散变成连续罢了。但数理统计不太一样，会给你带来有很多有趣的结论，比如，Stein's paradox 和 James-Stein Estimator。高斯分布是最重要的，不管在什么意义下，它的身影出现在信号处理、傅里叶分析、随机过程、优化的各个角落，常常摇身一变以协方差矩阵、核函数的形式出现在世界各地。你可听闻过已知均值和方差最大熵分布的传说，高斯混合模型下跃动的身影。

你开始进一步剖析面向对象的编程范式。之前都是用简单的控制流，现在转化为封装继承和抽象，但多态应用在小规模项目的时候你很难体会到他的优点，只有在一个几十万行的项目，一个基类拥有数千个子类的时候你才会感受到他的威力。模板，元编程，boost、absl 纷纷映入你的眼帘，计算机科学、可计算理论向你展示了冰山一角就足够你震撼许久。你可能听到 rust、lisp，可不要贪多。一定要跟着写，而不是只在脑子里过。现在你可以同时可以看浙大的数据结构慕课，它只要求 c 语言作为前置课程，每一节课设置了小作业和示例代码，这是很数据结构与算法分析的初印象。你发现 C 真是个麻烦的家伙，你说怎么会有 Python 这么好的语言，不用担心类型，自带成吨的数据结构。但它对于多人协作、大型项目简直就是灾难，胶水语言的另一个坏处是屏蔽了所有的底层实现，你现在逐渐对封装和抽象感到恐惧了，你害怕黑箱中会跳出什么梦魇。与之同时，你进一步探索 LLM 辅助编程的可能性，把 cursor，windsurf 和 copilot 使了个遍。永远牢记于心，不能丢掉自己的编程能力，只有你知道怎么写但不想写，才能让它写，否则你都不知道自己是怎么错的。vibe coding 在可预见的未来还不会取代工程师，他无法想象出未见之物，这也是人的局限性。

### 电路原理与物理

专业课会开始学习大学物理、化学，如果是制造、材料、测控、过程相关的学科，化学也要好好学，它本质上也是物理，不过加入了很多经验规则需要记忆，这是因为：more is different。保持物理直觉很重要，电路原理大概也开课了，这是你在本科第一次接触应用数学，你需要逐步培养建模的直觉，电路中有大量假设或者结论要求你记忆并使用，尝试分析使用这些定理的动机，推导他们的原理（利用微积分和线性代数的知识），比如集总参数假设、基尔霍夫定律。Gilbert Strang 在 mit18.06 的第十二课从图的矩阵表示推导得到了基尔霍夫电流和电压定律，图的拓扑结果可以通过代数方程描述，多么美妙的连接。你开始发现，仅仅记忆公式是不够的，你尝试寻找公式背后的"为什么"，人们尝试解决什么问题？这种形式的定义是如何得到的？打破砂锅问到底，最后你恐怕会回到《自然哲学的数学原理》，这是一种"未知最小化"。虽然还原主义不太妙，但这代表着你对庞大系统的原子认知，一种极致的好奇精神。我花了四年时间才破解了傅里叶变换的秘密，这是稍后不得不提的故事。

### 竞赛实践与计算机系统

此时你应该在某个学科竞赛的小团队里打磨了不少秘笈，里头的学长学姐对你做了培训和教学。如果是师徒绑定制，那么一有问题就去骚扰你的学长。你会提早接触一些"进阶"的内容，写个 yolo 识别各种物体啦（大概率是垃圾~），图像分割找病灶、行为识别判断是否摸鱼睡觉、语音识别和语音转文字、2d lidar slam 智能小车，抑或是用大模型做各种各样的下游应用，在专用数据集上 fine tunning。是时候一点一点把所学知识用到实践中去了，哪怕现在你学的东西大部分与实用都还差那么一丁点距离。和控制的核心一样，学习的关键是"反馈"机制。理论基础-实践-从结果中总结与反思-学习更多理论知识-完成闭环。你可以在笔记系统中新增一个模块：日记。定期的自醒与内窥是快速迭代的必要手段。把过剩的表达欲和分享欲倾泻在日记上吧。

知道了自己的不足，你打算接着精进自己的计算机能力，《CSAPP》早已滚瓜烂熟，你对操作系统的了解也更上一层楼。你加入了不少 linux 社区，对微内核颇有微词。你翻来覆去干了好几个发行版，最终还是回到了 ubuntu 2004 server，不过你的终端非常 fancy，这要归功于 zsh 和 ohmyzsh（下载它们！配置起来！）。你已经玩腻了 docker/wsl，k8s 里躺着一堆停止的容器，docker compose 对你来说信手拈来。你定制了一个 linux shell command cheat sheet 鼠标垫，有时候也在 linuxcmd 上查找一些生僻的指令，更多时候你使用 tldr 快速查找可能的解决方案，这是 man 的绝佳替代。不过你对这一切感到繁琐，这些工具性的指令是否可以通过自然语言描述？

准备好迎接光荣进化了吗？你需要更加现代化的终端，我们称之为——终端模拟器，加入鼠标的支持。这就是 Warp，一款使用 Rust 和 LLM 加持的超级终端：Warp the AI Terminal。你也可以尝试像运行记事本一样运行 shell 指令，vscode 有相关的插件，方便你将工作流分享给其他人。BTW，script 和 scriptreplay 允许你录制并重放在终端中的操作。在指令行键入自然语言，warp 会自动识别当前的工作模式，判断进行指令执行还是 LLM 提问。你不再需要记忆复杂的指令和参数，也不用手写烦人的循环和正则表达式，把一切交给 LLM 吧！Agent 模式还能自动 plan 复杂流程并分解执行，前线的 LLM 只要不断输出 token 就可以了，而后方的你需要考虑的可就多了，比如是先喝茶再按下 enter 还是先按下回车再 drink tea？

### 嵌入式进阶

高级一点的板子早就在购物车恭候多时，你趁着某个购物节将其一扫而空，stm/gd/esp/TI32/rockship 皆为池中之物。你对 linux 的了解更上一层楼，裁剪内核、将新的驱动打个补丁，移植 Uboot 如探囊取物。为了验证你的板子，你设计了一个三轴云台，在上面装上超声波传感器和红点激光器，琢磨着怎么整定 pid 的参数而不是 trial and error，明明只要两个自由度就可以指向空间中任意一个方向啊，我应该使用怎样的控制分解方式？动用你所有的知识储备，穷尽所有搜索的手段，你了解到旋转的表示有欧拉角、RPY 固定角，四元数，旋转矩阵，轴角表示。万向节思索困扰了你许久，四元数为什么能表示旋转，双倍覆盖又是什么意思？旋转矩阵竟然属于一个特殊的数学结构，李群，而轴角表示和旋转矩阵的李代数又是等价的。你还没回过神来，单位四元数竟然也是一个李群，其对应的李代数还和旋转矩阵相同，数学是一张密织的大网。想找到答案，恐怕你还得从流形、微分几何学起，在之前推荐的数学视频教程里找找答案吧，先把它们当成工具使用也不是什么坏事。

恐怖的事情是很多人在高中就已经到了这里，你在知乎和 github 见识了太多神人。不过不要对神人感到畏惧，你终将成为神人，观摩一下廖洽源高中时期的作品。你静下心思考程序的控制流，与 LLM 交互 coding 的工作流愈发熟练，有之前的基础手到擒来，控制、状态和数据寄存器在你面前一字排开任你宰割。你把抽象封装、层次化的思想推广到越来越多的场景下，这是完全通用的方法论。你已经厌倦了读写寄存器，改用标准库加速所有的操作，很快 HAL 成为了主流，大大简化了应用层代码的逻辑交互，在资源吃紧的地方你恰到好处地换用 LL。你还在 Github 上给 HAL/LL 提了不少 issues 和 pr，修了不少 bug。你现在可以谦虚地称自己为 STM32 core contributor，并把它加入用 latex template 编写的简历里。接下来是 rtos，你改造了一套自己的内存池，效率相比 heap4 竟然有 11.4514% 的提升，为此你专门写了一篇博客 post 在自己的主页上。你认为单片机和 desktop pc 的差距越来越小，裸机编程与操作系统的界限已然模糊，你正在跨越最后的界限。

### 深度学习与论文阅读

### 关于傅里叶变换的感悟

我在大二的时候转完专业，为了补齐之前漏下的课，一个学期需要上 7 门专业课。这包括积分变换和复变函数。这两门课的学习给我带来了巨大的困惑：积分变换的课本一上来就直接推导傅立叶级数并立刻缩小时间间隔将时域推向无穷，由此得到了连续的傅立叶变换。拉普拉斯变换则是直接复用前面的结论，一顿操作猛如虎然后把常用变换表丢到你脸上，你就背吧。我甚至不知道这些变换有什么用。我问老师傅立叶变换的虚部有什么物理意义？为什么实信号会在根本不存在的虚轴有分量？老师说，这一切都是为了方便计算。

我也试着自己将傅立叶级数和傅立叶变换的公式手推一遍，就像无数抄书抄公式的知乎回答一样，但这根本没有给我带来任何 insight，甚至都没有让我对他有熟悉的感觉，没有为我提供任何的直觉。这些定义和推导究其根本对我而言是凭空出现的，它的推导也只是从一个我不认识的样子变成另一个罢了。卷积、逆变换只是徒增我的心智负担，为什么有对偶的性质？更可怕的是，你很难向别人描述自己的感觉，你无法描述你不知道的事。

在我走遍了图书馆的所有角落和知乎的每个回答后，我终于发现，研究傅立叶变换应该在泛函的框架下进行，这里描述的数学对象是一个个函数，而不是数到数到映射，这是数到数到映射到数到映射。将函数视为无限维向量的视角将线性代数里你熟悉的工具引入希尔伯特空间，只需要一点（亿点？）小小的修改。傅立叶变换只不过是在做正交分解罢了，拉普拉斯变换则可以处理指数收敛的情况，推广到更多非指数增长的函数上。我为此懊悔了太久。也许我没有将我的疑惑描述清楚，因为在"顿悟"之后，它对我而言是非常自然的。希望这个小故事可以帮到你。个人看来，对工程师而言，建立数学直觉反倒比严格的公理演绎更重要。immersive math 里面提供了许多帮助你构筑空间几何概念的交互式可视化示意图。

### 模电与信号与系统

时间过得很快（一句废话）。这学期会安排模电的课程，其难度较大，不过好在他的方法论简单，就是局部线性化（小信号假设），你思考一番，这不就是仿射线性系统？在处理加法的时候可要小心了。模电领域有几本经典的大部头教材，如果你不信，可以登陆 libgenesis 和 zlib 上搜搜。如果之后打算做 IC 设计，模电务必要认真学好。另一个高质量开放电子书获取平台是 libretex，它和许多高校合作，提供了高清的 latex 制成的电子书，每年为国外学子节省上万的课本费（per person）。MIT，Stanford 的公开课基本把 slides 和作业都上传了，这是很好的课外补充。你的学校购买了许多学术出版商的数据库，但有时候你还是需要 scihub 作为补充，你创建了一个 scihub 域名扫描工具，每隔一段时间就更新有效地址。你本来还准备给它做种，但是没有地方放下那么多硬盘了。你现在去中心化、分布式的忠实粉丝。比特币背后的技术对你来说毫无难度，你（或许？）根本看不起炒币的人，认为用技术作恶将会被打下十八层地狱。但你对量化交易高看一眼，说不定自己可以加入某个私募投行巨头。

随之而来的是信号与系统，你打听到奥本海姆的小绿书是这门课的通关秘籍，转而沉浸在连续离散傅里叶级数和变换的对偶和对称关系中不能自拔。卷积终于向你展示了它的最高奥义，你这下明白冲激函数定义中无穷大的良苦用心了——分解、原子化，转化为方便分析的形式，仍然是分层、抽象和封装的方法论！你对泛函也有了更深入的理解，终于可以从集合、映射和运算出发理解数学法则。函数是数到数的映射，泛函是函数到数的映射，那泛函的泛函是什么？线性代数学习的对偶空间再次派上用场：有限维空间的双次对铺和原空间存在自然同构映射！从范围更大的数学空间里看待问题可以给你更有洞察力的结果，比如典型的，以无限维向量的视角看待函数，变分里的极值条件和 KKT 竟有惊人的联系！你有点感慨，学海无涯你只能望洋兴叹，甚至无法望当今最前沿数学研究之项背。

### 竞赛负责与控制理论

你应该在竞赛团队中作为项目负责人带领一个小组攻坚克难了。你会解决大量的工程问题，并在反思中尝试分析它为什么出问题、应该如何做更合理的假设。信号与系统、最优化、非线性优化理论纷至沓来，给你很多启示，告诉你工程的一些问题为什么行不通。噫，怎么不先介绍线性控制和线性系统？这是因为非线性是更一般更普遍的情况，线性系统先入为主将会给之后的学习带来可怕的思维定势，你认为所有系统都存在传递函数，当然你不会手绘老掉牙的跟轨迹，这东西应该和试卷一起丢到垃圾桶里。分析稳定性就用 Bode gram 或者 Nichols plot，最多加上 nyquist curve 把 disk Margin 画出来就万事大吉，但你不知道的是 LTI 的假设过于局限，一旦不是 L 或者 I 变成 V 这一切将归于尘土。MIMO 和非线性在过去的很多年都没有显著进展，大部分 control scientist 沉迷于做数学题。

### 解决工程问题的思维

你的数学基础扎实程度和广度应当远超大部分同学，有时候甚至萌生出一丝的蔑视。千万不要让它占据你的心灵，物理系、数学系的同学会把你秒成渣，后者是渣都不剩。解决工程问题需要很多经验，但不能只考经验。举个例子：

> 如果两个高速旋转的电机在启动之后在不远处的 IMU 数据中产生了拍现象，你会如何解决这样的低频扰动？
>
> - 第一时间你会想到滤波，但是使用一个截止频率这么低的滤波器，你的 IMU 算是废了。
> - 陷波器？那你得 IMU 岂不是无法跟踪这个频率的信号了？更别说通带的频率响应肯定也会受到影响。
> - 机械工程师说，加个减震垫，别用刚性连接不就得了，直接把这个频率的模态干掉。拜托，你已经没有时间重新设计结构了！
> - 加个 IMU 在更远的地方，随之而来的问题是系统变复杂可能带来的鲁棒性下降，你还要做时间同步和数据切换。
> - 不如就用锁相环（PLL）跟踪预先已知的干扰频率，在线估计出相位实现自适应陷波，直接消减对应的幅度？
> - 神人队友告诉你，与其估计它，他更相信 model-based，能不能根据电机模型和设定的参考速度直接计算出拍的相位，然后从 IMU 的原始数据减去这个部分？可是 IMU 内部都自带一级抗混叠滤波器，你会误伤有效信号的。

### 四旋翼与控制进阶

四旋翼飞行器的动力学和模型都过于简单，前馈 + 基于模型的 PID 已经把稳态误差治得死死的，在添加轨迹规划模块之前，LQT 并不会给系统带来额外的增益。你现在知道 LQR 和 MPC 的争论可以终止了，因为这里有离散/连续-有限/无限时域的区别，别人在苦恼如何手推得到 algebra ricatti equation 的时候，你在思考应该如何求解 DRE。你觉得四旋翼没法给你带来更多提升，调完了 PID 顺便学了一个内模控制，这是积分真正的精华。为了提高 PID 各个环节的性能，你参考了 PID Lib 优化，使用了积分分离、微分先行、微分滤波器和二次积分等 trick，并且用传递函数的方式分析了这些技巧的数学表达，自认在 pid 方面已无敌手。ADRC 看起来是个不错的工程改进，但是积分串联型的假设有点太强了，而且，总扰动把系统建模误差和不确定性也包含进去，这真的是可以估计的吗？

### 凸优化与数值分析

选修课的殿堂已经开门，很开门兄弟。你选（不得不选）凸优化和数值分析，它们非常实际，非常重要，不管有没有用能不能降低难度，几乎所有问题的 formulation 都可以统一到优化框架下，"最"是所有人一生的追求。boyd 的《凸优化》是脍炙人口的经典，知乎上大量的反刍和抄书笔记肯定能帮你学好他，别忘了用 matlab 或者你最爱的 mathematica 把里面的例子和作业动手跑一跑。

凸优化给了你获取最值的理论工具，实际在计算机上实现代码时，数值精读、溢出、病态条件等都是你需要考虑的问题，工程师就是要在有限的条件下最大程度地实现你的目标。推导一阶、二阶和拟牛顿法都没什么大不了，泰勒展开有手就行。关键是背后的理论分析，收敛速度、步长选取。动量法和共轭梯度都需要用到之前计算的结果，怎么实现可以最小化内存的占用和拷贝次数？矩阵运算要尽可能减少中间操作的数量，运算顺序也会极大的影响效率，你甚至需要考虑缓存能否放得下整个矩阵，内存读取的耗时和 cpu cycle 的数量级的差距。由于一些矩阵过于巨大，你在迭代法内部需要再嵌入一个迭代，来求解原本用于求解增量的先行方程组。消元法的顺序竟然能影响求解所需的次数，矩阵分解在这里大显身手。怎么又快又好地近似 hessian 同时避免求逆？万一这附近 hessian 不整定咋办？

这也是为什么求解器被制作成商业产品，一个好的 solver 包年的价格数十万有它的道理，有些甚至按调用次数计价。Matlab 高速的矩阵运算背后是数百个精英级的数学家兼工程师日夜兼程的作品。

### 随机过程

你的好朋友选了随机过程，但你已经没有更多精力分配给其他的课程了。你大致浏览了随机过程的目录，丢给 LLM 总结之后得出结论，这就是以随机变量为输入的动态系统罢了。那你可要小心了，不确定性扮演的角色可比这重要得多，并不是人们了解得不够多，很可能世界的本质就是不确定的。没事，你只要会求解高斯过程回归就行。说到 GPR，学过 GMM 的你立刻想到，我是不是可以弄个高斯混合过程回归？这岂不是把空间和时间都拉满了，任意概率密度函数肯定在指定逼近精度下肯定都可以分解成数个 Gaussian Mixture Process 的组合！别想了，早有人做过这个课题，你看了眼论文，期望最大化和核函数的选择正如你的想象，有这样那样的缺陷，过多的参数需要整定。没事，DL 在这方面的应用仍然是一片蓝海。GP 在时间序列分析、金融工程、信号处理、连续时间 SLAM 和统计回归等领域都有不同程度重要的作用。

### 通识能力

你对计算机的应用可谓挥洒自如，给团队里的新生培训时帮他们解决环境问题、分析 bug，在众人羡慕的目光里，你终于在与 gpt 的竞争中略微取胜。不论聊电脑配置、先进制程芯片、显卡，GPGPU，云计算，还是 cmos，各种消费/工业电子产品，你都能滔滔不绝说个不停。geekbay 的新视频你不屑一顾，你才是更专业的那个。谁家发布全新 cmos 了，rolling shutter 和 global shutter 的更迭，相位对焦和反差对焦的区别，如何拍摄 HDR 视频、ISO 和动态范围与宽容度的恩怨情仇这些数码区 KOL 都要费三分脑筋的玩意在你看来都是三岁小孩应该纠结的事情。不过你可能对更深层次的基础仍是未知全貌，你现在也不大在乎了，专业的事交给专业的人来做，让材料科学和物理学家操心吧。你能以平静的心态对待所有尖锐刺耳，与你相左的观点了。

你认为一切的消费电子、工业产品都无比熟悉，几秒内就可以绘出它们的架构图和工作流程，一个个金字塔抽象结构在脑中展开，意识随着数据流动。控制流在抽象层的边缘摆动，切换引导着数据的流动。虽然其中的几个专业词汇或者缩写让你摸不着头脑，但是只要让 LLM 稍加解释你就能打通所有流程。你对技术细节的把控堪称完美，绘制如此框架的能力已经内化。
